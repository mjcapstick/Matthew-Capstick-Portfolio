## Machine Learning
[Predicting Serverity of Alzheimer's in Patients](https://github.com/mjcapstick/Matthew-Capstick-Portfolio/blob/main/Alzheimer's%20model.ipynb) : This notebook was the first of my machine learning projects where we developed a model to predict Alzheimer's scores in patients 24 months after an initial diagnosis, using lasso regression we achieved an R-squared of 0.779 and RMSE score of 0.8607 on the test data. We received useful feedback on how to structure our analysis in a more understandable way which we implemented on our second project detailed below. \
[Default Payments in Mortgage Repayments](https://github.com/mjcapstick/Matthew-Capstick-Portfolio/blob/main/Default%20model.pdf) : This notebook details our process of developing a default payment model, we compared four algorithms SVM, decision tree, random forest and KNN. We justify our final choice and procced to tune hyperparameters to obtain our final model. Using SHAP values we could interpret the model and learn how various features contributed to the likelihood of defaulting on payments. \
[How Novice Players Can Improve Their ELO Rating](https://github.com/mjcapstick/Matthew-Capstick-Portfolio/blob/main/Chess_Project.pdf) : In my Final undergraduate project I developed a machine learning model to understand how Chess players can utilise different time formats to improve efficiently. Using open-source data from [lichess.org](https://database.lichess.org/) and the computational power provided by [eddie](https://digitalresearchservices.ed.ac.uk/resources/eddie) I could train my model for a range of player profiles.  \
[Labelling Heartbeats in ECG Recordings](https://github.com/mjcapstick/ECG-CNN-model) : In this repository, I use a CNN to label heartbeats in ECG recordings as either normal or abnormal. The abnormalities can be due to a range of diseases and conditions which are visualised. The data was open-source from [PhysioNet](https://physionet.org/content/mitdb/1.0.0/) and provides annotations from cardiologists. I am looking to improve on this in the future using a more interpretable model (e.g., a decision tree or lasso classification).

## Optimisation
[Optimising Power Systems in Scotland](https://github.com/mjcapstick/Matthew-Capstick-Portfolio/blob/main/Optimising_Power_Systems_in_Scotland.pdf) : This technical paper discusses potential frameworks for optimising hydroelectric systems in general considering the stochasticity of weather and implications for biodiversity. Finally conducting a case study on Loch Morar to determine the effectiveness of the proposed formulation. \
[Smoothing Surgical Bed Census via Block Scheduling](https://github.com/mjcapstick/Matthew-Capstick-Portfolio/blob/main/Smoothing_Surgical_Bed_Census.pdf) : This report details how block scheduling can be implemented to reduce the peak occupation of surgical beds. Improving on results from “An Optimization Framework for Smoothing Surgical Bed Census via Strategic Block Scheduling” by Carnes et al. (2011), we applied our updated framework on data based on the Massachusetts General Hospital. Conducting our optimisation in Gurobi, details of our code can be found [here](https://github.com/LucasXingg/Topics_in_OR_Project).
